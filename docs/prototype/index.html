新增 / 编辑路由

支持 IPv4 / CIDR / 域名，自动计算掩码，提供 Pin 与验证选项。

预览命令 route -p add 39.96.205.182 mask 255.255.255.255 192.168.1.1 IF 13 metric 3
变化摘要
  • Metric：5 → 3（提升优先级，标记为 change）。
  • 目标保持 39.96.205.182/32，未变化时 Apply 自动跳过。
  • 网关保持 192.168.1.1，Diff 中显示为“skip”。
  • CIDR 与掩码联动，默认以 IP/CIDR 形式维护永久路由。
  • 网关需与接口同网段，不通过时标红并禁止保存。
  • 勾选 Pin 后写入配置 pinned 字段，离线应用仍可使用固定 IP。

单条路由应用

仅对当前路由执行 Apply，避免影响其他条目，同时保留 Diff 记录以便审计。

目标 39.96.205.182/32
接口 / 网卡 vEthernet (mychange) · ifIndex 13
操作类型 change（Metric 调整）
变更内容 Metric：5 → 3；其他字段保持不变并自动跳过。
将执行 route -p add 39.96.205.182 mask 255.255.255.255 192.168.1.1 IF 13 metric 3
  • 单条 Apply 会记录至审计日志，并生成局部快照。
  • 若执行失败，可立即调用回滚功能恢复至前一个状态。

导入向导

支持 CSV / JSON 导入，自动校验字段、冲突策略与启用状态。

  1. 选择文件类型（CSV、JSON），上传或粘贴内容。
  2. 解析预览：展示字段匹配、CIDR 合法性、重复项提示。
  3. 选择冲突策略：覆盖 / 追加 / 跳过，支持仅导入启用项。
  4. 导入完成后可立即 Apply 或仅保存至编辑态。

导出配置

导出当前 Profile 或选中条目，支持 JSON / CSV，并附加审计信息。

快照与回滚

Apply 前自动生成系统路由与工具配置快照，支持差异对比与回滚。

  • 回滚采用 LIFO 顺序执行记录的反向命令。
  • 支持导出快照 JSON 用于线下审计或灾备。
